## Critical Issues (Blocking Functionality)

### 1. **Cloud Storage API Misuse** ⚠️ CRITICAL
**Problem:** Code attempts to instantiate cloud storage as objects:
```lua
-- INCORRECT (current code)
self.cloud_storage = DropBox:new{ server_setting = storage_config }
self.cloud_storage:uploadFile(...)
```

**Reality:** KOReader cloud modules use static methods:
```lua
-- CORRECT (what we need)
DropBox:uploadFileFromContent(url, password, content, callback)
WebDav:uploadFile(url, address, username, password, filepath, callback)
```

**Impact:** All sync operations fail with "attempt to call nil value" errors.

---

### 2. **MultiInputDialog API Misuse** ⚠️ CRITICAL
**Problem:** Code uses non-existent input types:
```lua
-- INCORRECT (current code)
{
    text = _("Enable Super Sync"),
    input_type = "check",    -- DOES NOT EXIST
}
```

**Reality:** MultiInputDialog only supports text input fields.

**Impact:** Settings dialog will crash or display incorrectly.

---

### 3. **Cloud Storage Settings Structure Wrong** ⚠️ CRITICAL  
**Problem:** Code treats settings as hash table keyed by name:
```lua
-- INCORRECT (current code)
for server_name, server_config in pairs(cs_settings) do
```

**Reality:** Settings stored as indexed array under `cs_servers`:
```lua
-- CORRECT structure
{
    ["cs_servers"] = {
        [1] = { name = "My Dropbox", type = "dropbox", ... },
        [2] = { name = "My WebDAV", type = "webdav", ... }
    }
}
```

**Impact:** Cloud storage selection fails; no providers found.

---

### 4. **ProgressWidget Misuse** ⚠️ HIGH
**Problem:** Uses low-level ProgressWidget instead of ProgressbarDialog

**Impact:** Progress display doesn't render properly on e-ink devices.

---

### 5. **Document Close Event Registration Invalid** ⚠️ HIGH
**Problem:** Uses non-existent event pattern:
```lua
-- INCORRECT (current code)
self.ui:handleEvent(Event:new("AddUpdateNotification", self.onCloseDocument, self))
```

**Reality:** Need proper event registration using KOReader patterns.

**Impact:** Auto-sync on document close never triggers.

---

### 6. **Missing FTP Support** ⚠️ MEDIUM
README promises FTP support, but only Dropbox and WebDAV are implemented.
